How does it work?

This tool lets you create an ISO image based on Ubuntu Server 22.04 in which you can customize different aspect of your system (e.g. packages and Docker images) and create an automated installation.

  • Insert you information like name, surname and email. This information will be used to create a unique identifier for your ISO file.

  • Choose which type of operating system you want. After you choose an option, you can add your preferred packages to your system.

  • Choose whether you'd like to install Docker on your system.

    If you choose to install Docker you'll be given the option to include your preferred docker images on your system. You'll have to insert the name of the images that will be pulled from the Docker Hub with the official Docker commands (i.e. docker pull <image name>).

    For this reason, the name of the image you want to include must correspond with the same name used to pull the image. For example, if you want the mongo and mongo express images on your system, just type "mongo" and "mongo-express" in the text area.

    You can refer to the Docker Hub for information on images name.

  • When the Generate ISO button appear, you can generate and then download your ISO.

  • Burn your ISO and start the installation. You'll be grated with the stock Ubuntu Server installation interface (i.e. Subiquity).

    Fill the information required by the installer like language, keyboard layout, network configuration and identity. The rest will be taken care automatically.

    Depending on the number of packages and images that you selected during the creation of the ISO this process may take several minutes. Please wait until a message stating that the installation has finished appear.

  • Restart your system and finish the installation.

    If you chose to install Docker on your system, some additional configuration will be automatically executed: a docker group will be created and the user created during the installation will be automatically added in order to give the permission required to pull images and create containers.

    In this step, the system will boot, and then automatically reboot in order for changes to take effect. Once the system has rebooted, when you log in, the docker images you chose will be automatically pulled from the Docker Hub.

    Like before, depending on the number of images you chose to pull, the size of them and your network connection, this process may take some time.
